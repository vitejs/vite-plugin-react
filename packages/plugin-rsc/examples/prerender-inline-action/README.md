# Prerendered inline server action replay

This example builds and executes an RSC prerender entry that writes a Flight payload containing an inline server action. A separate runtime-only RSC build replays that payload without importing or rerunning the page component, so invoking the action depends on the production server-reference manifest.

The page module also exports static metadata consumed by the SSR entry. This models frameworks that inspect route metadata in a non-RSC environment while rendering the route component only in the RSC environment.

## Build pipeline

1. Build and execute `entry.prerender.rsc.tsx`, writing `dist/flight/page.flight`.
2. Analyze the SSR graph, which imports the page metadata.
3. Build `entry.runtime.rsc.tsx`, which reads the stored Flight payload without importing the page module.
4. Build the client and final SSR bundles.

## Manual test

Start the development server:

```bash
pnpm dev
```

Alternatively, build once and start the production server:

```bash
pnpm build
pnpm preview
```

1. Visit `/cache` and invoke the inline action.
2. Restart the server without rebuilding.
3. Visit `/replay` and invoke the replayed inline action.

Delete `.flight-cache` to reset the runtime cache. Production prerender output is regenerated by `pnpm build`.
