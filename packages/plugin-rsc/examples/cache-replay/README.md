# Persisted Flight server reference replay

This example persists Flight payloads containing file-level and inline server references. It compares the default replay, which imports a server action in the RSC environment, with replay that preserves the reference. With preservation enabled, the action is imported only when the replayed form invokes it.

The cached inline-content route module also exports static metadata that the SSR entry renders as the document title on every request. This exercises frameworks that consume route metadata in a non-RSC environment while rendering the route component only in the RSC environment.

The production build uses three separate passes: it builds and executes a prerender entry that writes the Flight payload to `dist/.flight-inline-prerender`, analyzes the runtime RSC and SSR graphs, and finally builds a runtime-only RSC entry. The runtime entry replays the persisted payload without importing or rerunning the route component, so invoking its inline action depends on the server-reference manifest emitted by the final pass.

The framework files follow the starter example. The application routes own persistence and replay, while the framework only performs its normal request parsing, action handling, and RSC serialization.

## Manual test

Start the development server:

```bash
pnpm dev
```

Alternatively, build once and start the production server:

```bash
pnpm build
pnpm preview
```

1. Visit `/cache` and confirm the action is imported.
2. Stop and restart the server with `pnpm dev` or `pnpm preview`.
3. Visit `/read-cache`, then visit `/` and confirm the action is imported.
4. Restart the server again.
5. Visit `/read-cache-preserve`, then visit `/` and confirm the action is not imported.
6. Return to `/read-cache-preserve`.
7. Select **Invoke action** and confirm the action is imported and invoked.

The inline-action path is available at `/cache-inline` and `/read-inline-cache-preserve`. In production, `/cache-inline` replays the payload written to `dist/.flight-inline-prerender` during the prerender pass rather than rendering the component in the runtime bundle.

Do not change the source graph between development server restarts or rebuild between production server restarts. The persisted Flight payload contains server-reference IDs that must remain stable.

Delete `.flight-cache` and `.flight-inline-cache` to reset the example. The prerendered payload lives in `dist/` and is regenerated by `pnpm build`.
